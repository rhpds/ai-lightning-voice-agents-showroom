== MLflow

MLflow is used for experiment tracking and observability.

=== Helm Chart Deployment

The voice agent Helm chart includes MLflow as an optional component, enabled by default.

[.copypaste,source,bash]
----
helm install voice-agent ai-voice-agent/deploy/chart/
----

To disable MLflow:

[.copypaste,source,bash]
----
helm install voice-agent ai-voice-agent/deploy/chart/ --set mlflow.enabled=false
----

=== OAuth Proxy

By default, MLflow is protected by an OpenShift OAuth proxy sidecar. This requires users to authenticate via OpenShift before accessing the MLflow UI.

The OAuth proxy:

* Runs as a sidecar container alongside MLflow
* Listens on port 8443 (TLS) and proxies authenticated requests to MLflow on its internal port
* Uses OpenShift ServiceAccount-based OAuth
* TLS certificates are automatically generated by OpenShift via the serving-cert annotation

==== How it works

. A `ServiceAccount` is created with an `oauth-redirectreference` annotation pointing to the MLflow Route
. The `Service` is annotated with `service.alpha.openshift.io/serving-cert-secret-name` to auto-generate a TLS secret
. The `oauth-proxy` sidecar container handles authentication and proxies traffic to MLflow
. The `Route` uses `reencrypt` TLS termination and routes to the oauth port (8443)

==== Configuration

The OAuth proxy is configured in `values.yaml`:

[source,yaml]
----
mlflow:
  oauth:
    enabled: true
    image:
      repository: quay.io/openshift/origin-oauth-proxy
      tag: latest
      pullPolicy: Always
    cookieSecret: ""
----

To disable the OAuth proxy and expose MLflow directly:

[.copypaste,source,bash]
----
helm install voice-agent ai-voice-agent/deploy/chart/ --set mlflow.oauth.enabled=false
----

When OAuth is disabled, the MLflow Route uses standard `edge` TLS termination and routes directly to the MLflow HTTP port.
